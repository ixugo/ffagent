# FFAgent 转码处理全流程

> 本文档详细梳理「用户输入 → AI 分析 → ffmpeg 执行 → 结果交付」的完整链路，  
> 包含正常流程、错误处理、与 Claude Code Agent 循环模式的对比分析。

## 四、错误处理详解

### 4.1 各层错误及处理方式

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           错误类型分层                                       │
├─────────────┬───────────────────────────┬───────────────────────────────────┤
│    层级      │       错误类型             │         处理方式                   │
├─────────────┼───────────────────────────┼───────────────────────────────────┤
│             │ HTTP 请求失败              │ onError 回调 → 标记用户消息失败    │
│  前端       │ (网络断开 / Agent 未启动)   │ 用户可重试 (handleRetrySend)      │
│  (React)    ├───────────────────────────┼───────────────────────────────────┤
│             │ SSE 流未正常收尾           │ onIncomplete → 保留已收到的部分    │
│             │ (服务端崩溃 / 连接中断)     │ 内容 + 追加"响应未完整"提示        │
├─────────────┼───────────────────────────┼───────────────────────────────────┤
│             │ createSession 失败         │ markUserMessageFailed             │
│  API 层     │                           │ 用户气泡标红，可重试               │
│  (前端→Go)  ├───────────────────────────┼───────────────────────────────────┤
│             │ 保存消息到 DB 失败          │ slog.Error 记录，不阻塞流程       │
│             │                           │ (降级：消息丢失但不影响当前对话)    │
├─────────────┼───────────────────────────┼───────────────────────────────────┤
│             │ LLM 流式请求失败           │ SendMessage("AI服务暂时出了点问题")│
│  Agent      │ (API Key 错 / 网络超时)    │ SendDone() → return err          │
│  (RunAgent) ├───────────────────────────┼───────────────────────────────────┤
│             │ 参数解析失败               │ "参数解析失败: ..." 作为 tool_result│
│             │ (JSON 解析 tool args)      │ 计入连续失败 → 模型可自行修正参数  │
│             ├───────────────────────────┼───────────────────────────────────┤
│             │ 未知工具名                 │ "未知的工具: xxx" 作为 tool_result │
│             │                           │ 计入连续失败                       │
│             ├───────────────────────────┼───────────────────────────────────┤
│             │ 连续失败 >= 7 次           │ SendMessage(诊断提示 + 最近摘要)   │
│             │                           │ SendDone() → return nil           │
│             ├───────────────────────────┼───────────────────────────────────┤
│             │ 循环轮次 >= 25             │ SendMessage("已达处理轮次上限")    │
│             │                           │ SendDone()                        │
├─────────────┼───────────────────────────┼───────────────────────────────────┤
│             │ ffmpeg exit code != 0     │ "执行失败 (耗时 Xs):\n{stderr}"   │
│  ffmpeg     │                           │ 作为 tool_result → 模型分析重试    │
│  执行层     ├───────────────────────────┼───────────────────────────────────┤
│             │ exit 0 但文件不存在        │ 追加 ⚠️ 警告到 tool_result        │
│             │ (路径错 / sandbox 拦截)    │ "请勿声称已生成文件"               │
│             ├───────────────────────────┼───────────────────────────────────┤
│             │ ffprobe 执行失败           │ "ffprobe 执行失败:\n{stderr}"     │
│             │                           │ 计入连续失败                       │
│             ├───────────────────────────┼───────────────────────────────────┤
│             │ 执行超时 (10min / 30s)    │ context 取消 → exit code 错误     │
│             │                           │ stderr 携带超时信息               │
├─────────────┼───────────────────────────┼───────────────────────────────────┤
│             │ copyFile 失败             │ slog.Error + SendFile(缓存路径)   │
│  文件交付   │ (权限不足 / 磁盘满)        │ 降级：不复制，用户从缓存取         │
│             ├───────────────────────────┼───────────────────────────────────┤
│             │ 中间文件清理失败           │ slog.Debug 记录，不影响结果        │
│             │ (文件被占用 / 已删)        │                                   │
└─────────────┴───────────────────────────┴───────────────────────────────────┘
```

## 五、文件路径重写与交付机制

```
模型产出的路径                    系统实际操作                           用户最终看到
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

模型写 output.mp4               1. 取 basename → "output.mp4"
                                2. ensureUniquePath(cacheSessionDir, "output.mp4")
                                   → ~/Library/Caches/.../ffagent/{sessionID}/output.mp4
                                3. replaceLastOutputArg() 替换命令参数
                                4. ffmpeg 实际写入缓存目录


is_intermediate=true            记入 intermediatePaths[]                用户看不到
                                defer removeIntermediates() 清理         无 SendFile


is_intermediate=false           1. os.Stat() 确认文件存在
(最终成品)                       2. 有 userSourceDir?
                                   ├─ Yes → copyFile → SendFile(用户目录)  用户看到：
                                   │        删除缓存副本                    /用户/视频/output.mp4
                                   │                                       (可点击在 Finder 中打开)
                                   └─ No  → SendFile(缓存路径)             用户看到：
                                            保留缓存文件                    ~/Library/Caches/.../output.mp4

同名冲突处理:
  output.mp4 已存在 → output (1).mp4
  output (1).mp4 也存在 → output (2).mp4
  ...最多尝试 10000 次
```

## 六、SSE 事件协议

```
┌─────────┬─────────────────────┬──────────────────────────────────────────────┐
│  事件    │    方向              │    含义                                      │
├─────────┼─────────────────────┼──────────────────────────────────────────────┤
│ message │ Agent → 前端        │ LLM 流式文本片段（增量追加）                   │
│ status  │ Agent → 前端        │ 执行状态提示（"正在处理中，请稍等..."）         │
│ exec    │ Agent → 前端        │ 执行的命令（"ffmpeg -i ..." / "ffprobe ..."） │
│ file    │ Agent → 前端        │ 最终成品文件路径（可点击在系统中打开）          │
│ title   │ Agent → 前端        │ 会话标题（用户消息到达后异步生成，通过 SSE 实时推送）│
│ done    │ Agent → 前端        │ 本次对话处理完成                               │
├─────────┼─────────────────────┼──────────────────────────────────────────────┤
│ 编码     │ Base64              │ data 字段一律 Base64 编码，前端 atob 解码     │
└─────────┴─────────────────────┴──────────────────────────────────────────────┘

前端异常事件：
  onError       → SSE 请求失败或流中断（网络层错误）
  onIncomplete  → 连接关闭但未收到 done 事件（Agent 崩溃 / 超时断开）
```

### 8.2 正确的处理方案 vs 潜在问题场景

```
场景 A：简单转码（正常路径）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
  用户: "把这个视频转成 mp4" + 附件
  ✅ 预 probe → 模型分析 → execute_ffmpeg(is_intermediate=false) → 成品复制到用户目录

场景 B：多步处理（剪切+合并）
━━━━━━━━━━━━━━━━━━━━━━━━━━━
  用户: "剪掉第 2-5 秒，然后和另一个视频合并"
  ✅ probe → ffmpeg 剪切(intermediate=true) → ffmpeg 合并(intermediate=false) → 交付
  ✅ defer 自动清理中间文件

场景 C：ffmpeg 执行失败
━━━━━━━━━━━━━━━━━━━━━
  ✅ stderr 返回给模型 → 模型自动分析错误 → 调整参数重试
  ✅ 连续 7 次失败 → 终止 + 诊断提示
  ⚠️ 但如果模型反复犯同一个错误（如用了不支持的编码器），
     当前没有"已尝试过的错误策略"记忆机制

场景 D：exit 0 但无输出文件
━━━━━━━━━━━━━━━━━━━━━━━━━━
  ✅ os.Stat 检测 → 追加 ⚠️ 警告到 tool_result
  ✅ 告知模型"请勿声称已生成文件"

场景 E：文件复制失败
━━━━━━━━━━━━━━━━━━━
  ✅ 降级方案：SendFile(缓存路径) 而非用户目录路径
  ✅ 用户仍可从缓存目录找到文件

场景 F：LLM API 不可用
━━━━━━━━━━━━━━━━━━━━━
  ✅ SendMessage 友好提示 → SendDone
  ⚠️ 无自动重试或模型降级

场景 G：用户中途切换会话 / 关闭窗口
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ✅ sseCancelRef.current?.() 取消 SSE
  ✅ AbortController.abort() 中断 fetch
  ⚠️ Agent 侧 context 可能不会立即取消（HTTP 连接断开后 Gin context 会 cancel）
  ✅ defer removeIntermediates 仍会清理
```

## 九、数据持久化时序

```
时间线 ──────────────────────────────────────────────────────────────►

1. 用户发送消息
   │
   ├─ handleChatSSE: AddMessage(role=user, content, attachments)  ← 立即入库
   │
   ├─ go generateAndSaveTitle(sessionID, message)                 ← 立即异步协程
   │   │                                                            （与 RunAgent 并行）
   │   ├─ 检查 session.Title 是否为空
   │   ├─ 调用 LLM GenerateTitle                                   ← 通常 1-3 秒
   │   ├─ EditSession(title)                                       ← 落库
   │   └─ SendTitle(title) 通过 SSE 推送                            ← 前端侧栏立即更新
   │
   ├─ RunAgent 开始执行... (耗时 N 秒到数分钟，与标题生成并行)
   │
   ├─ RunAgent 结束，SendDone()
   │
   └─ handleChatSSE: AddMessage(role=assistant, content, files)   ← 完成后入库
```

> **注意**：assistant 消息的 `content` 来自 `sseWriter.fullContent`（拼接所有 SendMessage 文本），  
> `attachments` 来自 `sseWriter.collectedFiles`（仅含 SendFile 推送的成品路径）。  
> 工具执行的中间过程（exec 命令、status 状态）**不会**持久化到数据库。
